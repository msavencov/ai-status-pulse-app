# ADR-003: MCP для чтения, CLI для записи

**Дата:** 2026-03-16
**Статус:** Accepted
**Теги:** `devops`, `tooling`

---

## Контекст

Railway MCP сервер (`@railway/mcp-server`) - обёртка над Railway CLI. При использовании обнаружены ограничения:
1. **Типизация:** массивы и числа передаются как строки — `set-variables` и `deploy-template` ломаются
2. **Деструктивные операции:** `delete`, `restart` намеренно исключены из MCP
3. **Batch-операции:** установка нескольких переменных за раз не работает

Vercel MCP (`mcp.vercel.com`) - полноценный API без таких проблем.

## Решение

Паттерн **"MCP для чтения, CLI для записи"** для Railway:

| Тип операции | Инструмент |
|-------------|-----------|
| list, get, check status, logs | MCP (structured output) |
| set variables, deploy, delete, restart | CLI через Bash |

Для Vercel — MCP для всех операций (работает стабильно).

## Последствия

**Плюсы:**
- 100% покрытие функционала (CLI покрывает всё что не может MCP)
- Structured output от MCP для read-операций
- Claude Code автоматически выбирает правильный инструмент

**Минусы:**
- Нужно знать и помнить когда что использовать
- Railway CLI должен быть установлен и авторизован глобально
- Нет единого интерфейса для всех операций
