# ADR-004: Убрать signup, только login

**Дата:** 2026-03-16
**Статус:** Accepted
**Теги:** `frontend`, `product`

---

## Контекст

FastAPI Full-Stack Template включает полный auth flow: signup, login, recover-password, reset-password. StatusPulse - сервис мониторинга с админкой. Публичная страница (`/`) не требует авторизации. Админ-панель (`/_layout/*`) - только для авторизованных.

Вопрос: нужна ли самостоятельная регистрация пользователей?

## Решение

**Убрать signup, recover-password, reset-password.** Оставить только login.

Пользователей создаёт администратор:
- Через API (`/docs` Swagger)
- Через MCP серверы
- Через админку (`/_layout/admin` - управление юзерами, superuser only)

## Последствия

**Плюсы:**
- Меньше attack surface (нет открытой регистрации)
- Проще UI - одна страница логина
- Контроль над тем кто имеет доступ к админке
- Меньше кода для поддержки

**Минусы:**
- Нельзя добавить юзера без доступа к API/админке
- Нет self-service восстановления пароля (нужно сбрасывать вручную)

**Реализация:** `docs/backlog/003-frontend-customization/`
