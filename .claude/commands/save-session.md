# Save Session History

Создай детальную выжимку текущей сессии (чата) и сохрани в `session-history/`.

## Что делать

1. **Проанализируй весь чат** от начала до текущего момента
2. **Создай файл** `session-history/YYYY-MM-DD_HH-MM_short-slug.md` (дата и время старта сессии)
3. **Заполни по формату ниже**

## Формат файла

```markdown
---
date: YYYY-MM-DD
time: HH:MM
duration_approx: ~Xh Xm
participants: user + claude
---

# [Краткое название сессии — 5-10 слов]

## Цель сессии
[1-2 предложения — что хотели сделать]

## Что сделали

### [Блок 1: название]
- Конкретное действие 1
- Конкретное действие 2
- Результат

### [Блок 2: название]
- ...

## Ключевые решения
- [Решение 1 — почему выбрали именно так]
- [Решение 2]

## Созданные / изменённые файлы
- `path/to/file` — что сделали
- `path/to/file` — что сделали

## Незавершённые задачи
- [ ] Задача 1 — что осталось
- [ ] Задача 2

## Ошибки и workaround'ы
- [Ошибка] → [Как обошли]

## Контекст для следующей сессии
[Абзац — что нужно знать чтобы продолжить работу. Включи: текущее состояние деплоя, какие сервисы работают, какие нет, что ждёт действий пользователя, важные URL, credentials, переменные.]

## Коммиты этой сессии
- `hash` — message
- `hash` — message
```

## Правила

- **Детально, но кратко** — следующий агент должен получить полный контекст без чтения всего чата
- **Конкретика** — URL, имена файлов, ID проектов, хеши коммитов
- **Незавершённое** — обязательно указать что осталось и почему
- **Credentials** — НЕ включать пароли и секреты, только имена переменных
- **Ошибки** — включить найденные ограничения (MCP, сервисов) — это контекст для будущей работы
